## 🎯 Usage Scenarios

### **Trial Subscription Flow**
1. User subscribes with discount code → Code validated and stored in metadata
2. Trial starts (7 days) → Full access with discount pending
3. Trial conversion → Discount applied to payment automatically
4. Payment processed → Success with discount tracking

### **Immediate Paid Subscription Flow**
1. User subscribes with discount → Applied immediately to first payment
2. Payment processed → Success with discount metadata

### **Recurring Discount Flow**
1. User subscribes with recurring discount → Applied to first payment
2. Renewal 1 → Discount applied, remaining uses decremented
3. Renewal 2 → Discount applied, remaining uses decremented
4. Renewal 3 → Discount applied, remaining uses decremented
5. Renewal 4 → Full price (discount exhausted)

### **Late Discount Application Flow**
1. User has existing subscription → No discount
2. User applies discount code → Stored in metadata
3. Next payment (conversion/renewal) → Discount applied automatically

## 🚀 API Endpoints

### **Apply Discount at Subscription**
```
POST /api/v1/subscriptions/business/{businessId}/subscribe
{
  "planId": "plan_basic_tier1",
  "discountCode": "WELCOME20",
  "card": {...},
  "buyer": {...}
}
```

### **Apply Discount Later**
```
POST /api/v1/subscriptions/business/{businessId}/apply-discount
{
  "discountCode": "UPGRADE25"
}
```

### **Validate Discount Code**
```
POST /api/v1/discount-codes/validate
{
  "code": "WELCOME20",
  "planId": "plan_basic_tier1",
  "amount": 949.00
}
```
