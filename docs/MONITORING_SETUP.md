# Monitoring Setup Guide

## Overview

RandevuBu Server uses **Grafana** and **Prometheus** for monitoring and observability. This is a minimal, lightweight monitoring setup that uses ~500MB RAM and is completely free.

## What's Included

### Essential Services (Already Configured)
1. **Prometheus** - Metrics collection and storage
2. **Grafana** - Visualization and dashboards

### What's NOT Included (Intentionally Removed)
- ❌ **cAdvisor** - Container metrics (not needed for basic monitoring)
- ❌ **Node Exporter** - System metrics (not needed, uses extra resources)

## Resource Usage

| Service | Memory Limit | CPU Limit | Notes |
|---------|-------------|-----------|-------|
| Prometheus | 512MB | 0.5 cores | Scrapes app metrics |
| Grafana | 512MB | 0.5 cores | Displays dashboards |
| **Total** | **~1GB** | **1 core** | Fits comfortably on 8GB servers |

## Starting the Services

### Development
```bash
# Start with monitoring
./start-monitoring.sh  # Linux/Mac
# or
start-monitoring.bat   # Windows

# Or manually
docker-compose -f docker-compose.production.yml up -d
```

### Production (EC2)
```bash
docker-compose -f docker-compose.production.yml up -d
```

## What's Monitored

Prometheus scrapes metrics from:
- **RandevuBu App** (`/metrics` endpoint) - Request rates, response times, error rates
- **Prometheus itself** - Self-monitoring

## Key Metrics Available

1. **HTTP Request Duration** - Response time percentiles (p50, p95, p99)
2. **HTTP Request Total** - Total requests by status code
3. **Error Rate** - 4xx and 5xx errors
4. **Request Rate** - Requests per second
5. **Active Connections** - Current active HTTP connections

## RequestId Tracing

All requests include a unique `requestId` that:
- Is generated by `requestIdMiddleware` at request start
- Is stored in async context for logger access
- Is included in response headers (`X-Request-ID`)
- Is consistent across all logs and error responses
- Allows full request lifecycle tracing

### Searching Logs by RequestId

```bash
# Find all logs for a specific request
grep "abc-123-def" logs/all/*.log

# Or in Docker
docker-compose logs app | grep "abc-123-def"
```

## Cost

- **Licensing**: FREE (100% open source)
- **AWS Cost**: $0 (runs on your EC2)
- **Storage**: ~200-500MB for metrics (8 days retention)

## Removing Monitoring (Optional)

If you don't want monitoring, simply remove from `docker-compose.production.yml`:

```yaml
# Comment out or remove these sections:
# prometheus
# grafana
```

Then restart:
```bash
docker-compose -f docker-compose.production.yml down
docker-compose -f docker-compose.production.yml up -d
```

## Troubleshooting

### Prometheus can't scrape app
```bash
# Check if app is running
docker-compose ps

# Check app logs
docker-compose logs app

# Verify /metrics endpoint
curl http://localhost:3001/metrics
```

### Grafana shows no data
```bash
# Check Prometheus targets
# Visit: http://localhost:9090/targets
# Should see "randevubu-app" as UP

# Check Grafana datasource
# Visit: http://localhost:3000
# Go to Configuration > Data Sources > Prometheus
# Click "Test" button
```

### High memory usage
```bash
# Check resource limits are applied
docker stats

# Reduce retention if needed
# Edit docker-compose.production.yml:
# Change: --storage.tsdb.retention.time=200h
# To: --storage.tsdb.retention.time=100h
```
